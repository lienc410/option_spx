# Q087 A3 — /ES HV Ladder 止损双口径审计（2026-07-05）

**问题**: 晋升回测（Q071，V2f+G6）验证的是 `V2F_STOP_MULT=15×`；实盘 Telegram 纪律（SPEC-086）是 2× 警告/3× 触发。**实盘在执行一个从未被回测验证的止损，回测验证的止损从未被实盘执行。** 失配起源：基础相位曾对齐 bot（STOP_MULT=3.0，注释明写"Matches bot trigger"），V2f 相位引入 15× 后晋升时无人调和。

## 2. 读法

1. **15× 在回测里是装饰性的**（26 年零触发）——V2f 的真实风控是 10% 止盈 + 5DTE 强平 + 趋势门 + VIX≥22 入场门。
2. **3× 纪律的历史代价 = 总 PnL 的 -34%**（$61k/26y），换来的只有单笔最差从 -$24k 截到 -$15k；**CVaR10 反而恶化**（-$7.4k→-$9.3k）——止损把本会回血的仓位锁定成 -2× credit 的实亏，尾部桶更挤。保险费贵且保错了层。
3. 2024+ 五笔两口径完全相同（近期未触发）——失配的代价集中在高波动时代。
4. **模型缺口如实标注**: 回测按日收盘 mark（BS@VIX），实盘 TRIGGER 是日内监控——真实 3× 触发频率高于模型，实盘口径的真实代价**大于**上表估计；同时日内跳空/流动性风险也是 15× 口径未建模的暴露。两个方向的未建模量都存在。

## 3. Verdict 与 PM 决策点

**失配必须消除**（治理层结论，无争议）。方向是 PM 风险姿态决策：

- **方案 a — 实盘对齐已验证口径**：Telegram TRIGGER 撤销或改 15×（保留 2× WARNING 作情报）。取回 ~1/3 历史期望，代价是接受 -$24k 级单笔（发生在已验证的 26y 分布内）
- **方案 b — 重新按 3× 口径晋升**：接受 $120k/26y 的低配版本，实盘纪律不变，回测账面与实盘一致
- 5× 折中**不推荐**——事后切点选择，违反预注册纪律

**流程**: 本 verdict 与 A1 打包送外审 → 外审后 PM 决策 → 中选方向走 SPEC。实盘现状在决策前不变（当前无持仓风险敞口变化）。
