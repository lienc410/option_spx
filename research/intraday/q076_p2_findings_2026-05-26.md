## 1. Selector 当前行为的关键发现（PM 提的语义问题）

PM 问："低 IVP 在 baseline selector 里是 entry filter 还是 close trigger?"

**答**：production 设计是 **actionable close trigger**。证据在 `strategy/state.py:324-341`：

```python
def get_position_action(new_strategy, is_wait, strategy_key):
    current = read_state()
    if current is None:
        return "WAIT" if is_wait else "OPEN"     # 无持仓 → 只 entry block
    if is_wait:
        return "CLOSE_AND_WAIT"                  # 有持仓 → 强制 close
```

我之前 replay 数据里所有低 IVP 行都显示 `WAIT`（不是 `CLOSE_AND_WAIT`），是因为 `read_state()` 读 production live position 文件，replay 里为空 → 返回 `WAIT`。**真实持仓场景下 selector 设计就是把"低 IVP"当作 close 信号**。

→ A2a 复现 production 行为；A2b 是对该设计的 deviation（PM 提议）。

## 2. 六个变体结果

PM 成功标准：**flips ↓ ≥ 50% AND ≤3h episodes ↓ ≥ 75% AND EOD agreement ≥ 90%**

| Variant | Flips | ↓vs base | ≤3h ep | ↓vs base | EOD agree | Round trips | **Verdict** |
|---|---|---|---|---|---|---|---|
| baseline | 15 | — | 4 | — | 100% | 7 | — |
| **A2a** (production close-trigger) | 9 | -40% | 1 | -75% | 95.2% | 4 | **FAIL** (flips) |
| **A2b** (entry-only deviation) | 7 | **-53%** | 1 | **-75%** | **90.5%** | 3 | **✓ PASS** |
| B (scheduled only) | 11 | -27% | 2 | -50% | 100% | 5 | FAIL |
| **A2a + B** | 7 | -53% | 1 | -75% | 90.5% | 3 | **✓ PASS** |
| **A2b + B** | 7 | -53% | 1 | -75% | 90.5% | 3 | **✓ PASS** |

### 关键观察

1. **A2a 只 -40% flips** 因为它**加回了低 IVP 的 close trigger**。当 IVP 从 ~40 跌到 ~35（PM 持仓期间），强制 close → 多了 close-and-reopen 事件。**production 现有设计本身贡献了 churn**
2. **A2b -53% flips** 因为它把低 IVP 当 entry filter，不强制 close 现有仓位（5-21, 5-22 两天保持 BPS hold 而非关单）
3. **A2a + B = A2b ≈ A2b + B**（都是 3 round-trips, 7 flips, 90.5% EOD agreement）。B 在 A2a 上"抹平"了低 IVP 抖动 close 因为只在 10:30 / 15:30 评估，跳过了边界 noise sample
4. **B 单独不够**：cadence-only 不能去掉跨小时的真实 IVP 抖动

## 3. EOD disagreement 的 2 天分解（A2b vs baseline）

| Date | VIX EOD | IVP EOD | baseline | A2b | 说明 |
|---|---|---|---|---|---|
| 2026-05-21 | 16.68 | 38.6 | Reduce / Wait | **Bull Put Spread** | A2b 因 prev=BPS + IVP > 35 继续 hold |
| 2026-05-22 | 16.81 | 41.4 | Reduce / Wait | **Bull Put Spread** | 同上 |

**这两天的 disagreement 本质就是 A2b 的设计意图**：让低 IVP 持仓继续 hold（收完 theta），不强制 close。

在 production 当前设计下这两天会出现：
- 5-21 关闭 BPS
- 5-22 上午 IVP 41.4 → 仍 wait（不重开）
- 5-22 下午 / 5-26 IVP 回升至 41-46 → 重开

A2b 把这种"低 IVP 关-高 IVP 开"循环消除了。

## 4. 三个 PM 决策路径

### 路径 1: **A2b 单独**（strategy-semantics change）
- 改 selector.py：让低 IVP 仅作为 **entry block**，不返回 `CLOSE_AND_WAIT`
- 优点：最简洁；一次改动覆盖所有触发场景；理论上对 theta 收益更友好（不在已收 premium 后强制让步）
- 缺点：**改变 strategy semantics**——这是 SPEC 级修改，需要审视：
  - 19y backtest 里有多少笔 BPS_HV 是因为 `CLOSE_AND_WAIT` 提前关掉？这部分 P&L 改变？
  - Aftermath gate 之类的复合 rule 会不会因此受影响？
  - 这是一个**单独的 SPEC**（"Low-IVP entry-only"），不应该和 Q076 churn 治理混在一起

### 路径 2: **A2a + B**（governance-layer change）
- 不改 selector.py
- 只加两层 wrap：
  - 上层：scheduled evaluation 10:30 + 15:30 ET
  - 下层：双侧 hysteresis state machine（IVP 42-53 entry / 35-57 hold）
- 优点：**不动 selector**；纯 governance 工程；治理风险隔离
- 缺点：两层 wrap 复杂度；需要 frontend + bot 双侧实现

### 路径 3: **A2b + B**（两层都加 + selector 改）
- 同时改 selector semantics + 加 governance 层
- 优点：最稳；defense in depth
- 缺点：两件事一起做混淆 audit；不建议

### 我的推荐

**路径 2: A2a + B** — 理由：
1. **Q076 是 execution governance 问题**（PM 自己说），不是 strategy semantics 问题。修 governance 层是正确的 scope
2. A2b 的 "low-IV entry-only" 是一个**值得单独研究的策略问题**——19y backtest 会回答它对 BPS_HV/BPS 收益的影响，但**那是另一个 SPEC**，不应该塞进 Q076
3. **A2a + B 在数据上与 A2b 完全等价**（同 3 round-trips, -53% flips, 90.5% EOD）—— 在不改 selector 的前提下达到同样治理目标
4. SPEC 边界清晰：Q076 ⇒ governance wrap；low-IV semantics ⇒ separate future SPEC（如果 PM 想做）
