## 第二步：Option Quant Researcher 评价

### 总体判断

这个策略**值得加入下一步研究池，但不能直接照搬为主策略**。

它本质上不是一个单一策略，而是一个 **large-cap equity option income overlay**：

1. 核心底仓上卖 Covered Call。
2. 想接的股票下方卖 Cash-Secured Put。
3. 财报前利用高 IV 卖 OTM option，等待财报后 IV crush。
4. 对极端 overbought 个股做 opportunistic short / call selling。
5. 长期保持较高 cash buffer，例如 30%。

从量化研究角度，这里面有可研究的 alpha / carry 组件，但原帖表达方式过度营销化，缺少最关键的风险定义、仓位规则、tail risk 控制和真实 benchmark。

---

# 1. 策略里真正有价值的部分

### A. Covered Call on mega-cap core holdings

这个部分最稳，最适合研究。

对 MSFT、AAPL、AMZN、BRKB 这种高质量大盘股，Covered Call 本质是：

> 持有股票 beta + 卖出 upside convexity + 收取 implied vol premium

它适合的市场环境是：

| 市场环境 | Covered Call 表现 |
| ---- | --------------- |
| 横盘   | 好               |
| 缓慢上涨 | 好               |
| 快速上涨 | 跑输正股            |
| 小幅下跌 | 有保费缓冲           |
| 大跌   | 仍然承担正股亏损        |

真正要研究的问题不是“能不能收租”，而是：

> 卖掉 upside 后，长期 Sharpe / Calmar / drawdown / tax-adjusted return 是否优于简单持有正股？

对大资金账户来说，这个策略有现实意义。尤其是如果账户本来就长期持有这些股票，那么 CC 是一个 overlay，不是 standalone trade。

**结论：可以纳入研究。**

---

### B. Cash-Secured Put at desired entry price

这个部分也值得研究，但要非常严格控制 underlying selection。

Cash-secured put 的经济含义是：

> 卖出下跌保险，收取 premium；如果跌破 strike，就以目标价格接货。

它适合用在用户本来就愿意长期持有的股票上。比如 MSFT、AAPL、AMZN、BRKB 这类资产，逻辑相对合理。

但对 UNH、CAR、SNDK 这种 idiosyncratic gap risk 较大的股票，CSP 会有更大的 jump risk。

关键研究点应该是：

1. 卖 Put 的 strike 如何定义？
   例如 delta 10、delta 15、delta 20，还是基于 realized vol / support level。

2. 是否只在 IV rank 高的时候卖？
   如果不加 IV filter，长期 premium 不一定足够补偿 crash risk。

3. assignment 后如何处理？
   是长期持有、卖 covered call，还是 stop loss？

**结论：可以纳入研究，但必须只放在 high-quality whitelist 下。**

---

### C. Earnings IV crush short option strategy

这是帖里最有“策略感”的部分，但也是最容易被误解的部分。

他说：

> 财报前涨得多的，OTM sell call 多一些；跌得多的，OTM sell put 多一些。

这个逻辑可以理解为：

| 财报前走势 | 操作         | 隐含假设                        |
| ----- | ---------- | --------------------------- |
| 财报前大涨 | 卖 OTM Call | 市场过度乐观，upside 已 priced in   |
| 财报前大跌 | 卖 OTM Put  | 市场过度悲观，downside 已 priced in |

核心收益来源不是方向判断本身，而是：

> earnings implied move 通常被高估，财报后 IV crush 足够大。

这确实是一个可研究方向。但最大问题是 jump risk：

一个财报 gap 可以直接穿透 30% OTM strike。尤其是单名股票，历史上并不罕见。
如果裸卖 call，风险理论上无限；如果卖 put，风险接近持有股票下跌到零。

所以研究时不能叫“卖出期权收租”，应该定义为：

> Earnings Event Volatility Risk Premium Harvesting Strategy

推荐研究变体：

| 版本                     | 说明                                | 风险           |
| ---------------------- | --------------------------------- | ------------ |
| Naked OTM short option | 原帖类似做法                            | 风险最高，不建议直接实盘 |
| Cash-secured put only  | 只在愿意接货的股票上卖 put                   | 中等           |
| Covered call only      | 只在已有底仓上卖 call                     | 中等偏低         |
| Credit spread          | 卖 OTM option，同时买更远 OTM protection | 更适合系统化       |
| Iron condor            | 双边收 IV crush                      | 需要更精细风控      |
| Calendar / diagonal    | 交易 vol term structure             | 更复杂          |

**结论：可以研究，但推荐用 defined-risk spread 版本，不建议研究裸卖作为主方案。**

# 2. 策略中明显有问题的部分

### A. “RSI 99 不可能不跌”是危险表述

RSI 99 代表极端 overbought，但不等于一定下跌。

更准确的 quant 表达应该是：

> RSI extremely high may indicate stretched short-term momentum / mean-reversion probability, but the conditional distribution is highly regime-dependent.

极端强势股可能继续 squeeze。尤其是小盘股、低流通、hard-to-borrow、high short interest 股票，RSI 越高越可能继续逼空。

CAR 这种 trade 他说“800 多借出来，200 还回去”，如果是真的，这是非常典型的 high-risk idiosyncratic short trade。它不是稳定 option income 策略，而是 opportunistic short squeeze reversal trade。

**不建议作为核心研究方向。**

最多作为独立 event-driven case study。

---

### B. “30% OTM sell call 财报后归零”有幸存者偏差

财报前卖 30% OTM call 看起来很安全，但实际问题是：

1. 有些股票财报后真的可以跳涨 20%-40%。
2. 卖 call 的收益有限，亏损右尾很厚。
3. 一次极端 gap 可能吃掉多次 premium。
4. 如果没有买保护腿，长期风险收益分布很不对称。

这个策略看起来像“高胜率、低平均收益、偶尔巨大亏损”的 short vol trade。

它的研究重点必须是：

> max loss / expected shortfall / worst 1% event，不是 win rate。

---

### C. “Covered Call 不怕卖飞”只适合特定目标

Covered Call 最大的隐性成本就是卖飞。

如果一个账户的目标是：

> 稳定现金流 + 降低波动

那 CC 有意义。

但如果目标是：

> 最大化长期 total return

CC 经常会跑输 buy-and-hold，尤其在强 bull market。

所以不能简单说 CC 是“提款机”。它更像：

> 把未来上涨空间部分卖掉，换取当前现金流。

这不是免费午餐。

---

### D. “SP / CC 双向收租”容易低估相关性风险

在同一个 underlying 上同时做 CC + CSP，本质接近 short strangle / wheel strategy。

如果股价横盘，很舒服。
如果股价大跌，put 被 assignment，同时原有股票也亏。
如果股价大涨，call 被 assignment，upside 被截断。

这类策略的风险是：

> 收益像 fixed income，亏损像 equity crash。

研究时必须统一看总组合 exposure，不能把每一笔保费孤立看。

# 4. 和你现有 SPX / option strategy research 的关系

这个策略和你现在偏 SPX / index options 的研究有一个重要区别：

### SPX / index options

优点：

1. 单名 jump risk 低。
2. 流动性好。
3. European style，现金结算。
4. 税务处理通常更友好。
5. 更适合系统化回测。

缺点：

1. edge 更拥挤。
2. 收益来源更偏 broad short-vol / variance risk premium。
3. 很难靠个股财报 IV crush 获得额外机会。

### Single-name option income

优点：

1. IV crush 更明显。
2. 个股事件更多，机会更分散。
3. 可以结合 fundamental whitelist。
4. 适合已有股票底仓的 overlay。

缺点：

1. gap risk 大。
2. earnings event 难建模。
3. borrow、assignment、liquidity、tax 都更复杂。
4. 极易被幸存者偏差和社媒战绩误导。

所以它可以作为你的研究组合中的：

> Equity Option Income Sleeve

但不应替代 SPX 主线策略。

# 5. 建议给下一步研究的正式定义

可以把这个策略整理成一个 research spec：

## Strategy Candidate: Large-Cap Equity Option Income Overlay

**Objective:**
在高质量美股底仓或目标买入股票上，通过 covered call、cash-secured put、以及 earnings IV crush defined-risk structures，提升现金流和 ROE，同时控制 assignment、gap risk 和 portfolio drawdown。

**Universe:**
AAPL, MSFT, AMZN, GOOGL, META, NVDA, BRKB, WMT, COST, JPM, UNH 等高流动性 large-cap。
排除 hard-to-borrow、小盘、meme、低流动性、高 idiosyncratic jump risk 股票。

**Core Signals:**

1. IV Rank / IV Percentile
2. Earnings implied move vs realized post-earnings move
3. Pre-earnings run-up / selloff
4. Delta-based strike selection
5. Liquidity filter: bid-ask spread, open interest, volume
6. Portfolio concentration limit
7. Cash buffer requirement

**Candidate Trades:**

1. Covered call on existing holdings
2. Cash-secured put at target entry level
3. Put credit spread into earnings
4. Call credit spread into earnings
5. Iron condor only for selected stable names
6. Wheel strategy after assignment

**Risk Rules:**

1. No naked short call unless covered by shares.
2. No short put unless cash-secured or defined-risk.
3. Per-name max loss cap.
4. Earnings exposure cap.
5. Sector concentration cap.
6. Maintain minimum cash buffer, e.g. 20%-30%.
7. Stress test 2008, 2020, 2022, and single-name earnings gaps.

# 6. 我的最终评价

这套东西里面有真实可研究的成分，但原帖有明显的“卖课/社媒包装”味道。真正有价值的不是“一个月收多少权利金”，而是下面几个问题：

1. 收的 premium 相对于占用 capital 的 ROE 是多少？
2. 相对于 buy-and-hold 是否有超额收益？
3. 最大回撤和 tail loss 是否可接受？
4. 财报 short vol 的 crash loss 是否被充分计入？
5. 这个策略是否只是 bull market + high IV 环境下的阶段性好看？
6. 税后、滑点后、assignment 后是否仍然有效？

**建议：加入下一步研究，但只研究机构化版本：**

> large-cap whitelist + covered call / cash-secured put + defined-risk earnings short vol + strict cash buffer。

**不建议研究或复制：**

> 裸卖财报期权、RSI 99 做空、单名股票极端事件押反转、用“月收租金额”评价策略好坏。

这类策略对你的项目有参考价值，尤其适合作为 **idle BP deployment / equity income overlay** 的候选模块；但它不能替代你的 SPX 主策略，也不能用社媒式“收租”收益来判断优劣。
