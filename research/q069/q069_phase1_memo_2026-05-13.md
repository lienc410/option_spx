# Q069 Phase 1 — Smoothed IVP Gate

**Date**: 2026-05-13
**Owner**: Quant Researcher
**Status**: 完成；**5 个 smoothing 变体全部 fail strict dominance；但揭示了 fundamental meta-pattern**

---

## Question

Per 2nd Quant Q068 review framing：
> "未来要重开，应该是一个真正不同的研究假设，而不是继续在 hard gate 周边加小修小补。"

Q069 Phase 1：测试"修改信号输入"而非"修改 gate 逻辑"——对 IVP_252 做 short-window smoothing，然后应用同 55 阈值。

**Hypothesis**: 如果 jitter 来自 percentile rank-jump，short-window smoothing 应该能保留长期信号的同时减少 daily noise。

## Findings

### 1. Flip rate 完美按预期下降 ✅

| Variant | block_% | flip_rate | flip_flop_5TD |
|---|---|---|---|
| V0 raw | 43.2% | 7.37% | 219 (61%) |
| V1 SMA3 | 42.4% | 3.76% | 67 (37%) |
| V2 SMA5 | 42.5% | 3.18% | 42 (27%) |
| **V3 SMA10** | 42.2% | **2.14%** ✅ | 11 (11%) |
| V4 EWM α=0.3 | 42.6% | 3.10% | 50 (33%) |
| **V5 EWM α=0.1** | 41.1% | **1.66%** ✅ | 14 (17%) |

Smoothing 机械上有效——V3/V5 跨过 < 3% flip rate 目标。**Block 总比例几乎不变**（41-43%），证明 smoothing 没有 systematic shift gate，只是 reduce noise。

### 2. Worst trade 全部恶化 ❌

| Variant | Worst trade | Δ vs V0 |
|---|---|---|
| V0 raw | -$9,379 | — |
| V1 SMA3 | -$15,119 | **-$5,740** |
| V2 SMA5 | -$15,713 | -$6,334 |
| V3 SMA10 | -$15,713 | -$6,334 |
| V4 EWM α=0.3 | -$15,119 | -$5,740 |
| V5 EWM α=0.1 | -$15,119 | -$5,740 |

**根因 — Smoothing 引入 LAG**：IVP 真正 spike 时（如 GFC pre-Lehman），smoothed signal 需要数日才跨过阈值。这几天里，bad trade 已进场。

### 3. Alpha 单调流失

| Variant | Total | Δ Ann/yr |
|---|---|---|
| V0 raw | $81,581 | — |
| V1 SMA3 | $50,594 | -$2,027 |
| V2 SMA5 | $29,350 | -$3,205 |
| V3 SMA10 | $26,796 | -$3,435 |
| V4 EWM α=0.3 | $53,042 | -$2,256 |
| V5 EWM α=0.1 | $47,056 | -$2,018 |

**Pattern**：smoothing window 越长 → flip 越低 → alpha 流失越大。线性 trade-off。

### 4. Regime-conditional anomaly (V5 recent 2y)

| 窗口 | V0 raw | V5 EWM α=0.1 |
|---|---|---|
| Full 19yr total | $81,581 (36 trades) | $47,056 (25) → -$34k |
| Recent 2y total | $24,520 (14 trades) | **$23,153 (8) ≈ V0!** |
| Recent 2y worst | -$9,379 | **-$290** (!) |

V5 在 recent 2y 用一半 trades 赚到几乎同样的钱 + worst trade 几乎为 0。但全样本是 fail。

**这是与 Q068 P6 一模一样的 "recent regime works, full sample fails" pattern**。

### 5. Agreement analysis 揭示 smoothing 改变的方向

| Variant | TD 与 V0 不同 | Smooth allows + Raw blocks | Smooth blocks + Raw allows |
|---|---|---|---|
| V1 SMA3 | 5.36% | 149 | 112 |
| V3 SMA10 | 11.60% | 306 | 259 |
| V5 EWM α=0.1 | 12.87% | 363 | 264 |

**Smoothing 在双方向都改变 gate 决策**——不只是"少 block 几天"，是 systematic shift in timing。

## Meta-Finding：四方向研究统一 pattern

这是 Phase 1 最重要的发现：

| 研究 | 方向 | 主要 worst trade |
|---|---|---|
| Q063 Phase 5 | Multi-factor relax | n/a (rejected on PnL) |
| Q067 Phase 2 V1 hysteresis | Asymmetric gate | -$3,679 (但 bps_n=5) |
| **Q068 Phase 6 P6A/B/C** | **MA timing override** | **-$15,119 / -$15,713** |
| **Q069 Phase 1 (今天)** | **Smoothed IVP** | **-$15,119 / -$15,713** |

**Worst trade 一致 collide 到 -$15,119 ~ -$15,713**——这不是巧合，是 **same historical bad trade 被不同机制"延迟 block"放进策略**。

> **Fundamental 结论**：任何对 IVP gate 的"软化"修改（hysteresis / smoothing / MA filter / regime stop）都通过同一机制损害 worst trade —— **延迟 block 时机**。
>
> Threshold-based 修改的 trade-off 是确定性的：noise reduction 与 worst trade preservation 不可兼得。

### 这是否暗示 non-threshold-based 方向是唯一出路？

可能。Threshold gate 的核心问题：
- Hard threshold + 在 jitter zone 频繁穿越 → flip
- 任何 smoothing / asymmetry / filter → lag → 错过 risk spike → worst trade 恶化

**非 threshold 方法**（probability / Bayesian / continuous score）理论上能避免"在临界值附近 binary flip"，但工程复杂且 overfit 风险高。

## Next Step Decision Point

### 三个 path 选择

| Path | 含义 | 推荐度 |
|---|---|---|
| **A. Close Q069, accept fundamental limit** | 接受"threshold-based gate 不可能同时 reduce jitter 和 preserve worst trade"；Q063/Q067/Q068/Q069 四轮一致；不再 explore threshold variants | ✅ 强推荐 |
| **B. 启动 Phase 2 regime detection** | 定义 low-vol-clustered / transitioning / elevated 三 regime，每 regime 动态阈值。**但 2nd Quant 警告 "用 recent 2y 强行覆盖 full-sample fail 容易变成 sample chasing"，regime detection 本身就是高 overfit 风险** | ⚠ 风险 |
| **C. 直接跳 Phase 3 non-threshold-based** | Bayesian / probability framework。Phase 1 meta-pattern 暗示 threshold 修改是死路，可能 P3 是真正的出路。但工程量大（1+ 周），overfit 风险也高 | 🟡 高工程量 |

### Quant 推荐

**A**。理由：
1. 4 个研究方向全部失败，且**死因相同**（lag → worst trade）——这是 fundamental 不是"再试一个变体能解决"
2. Regime detection (P2) 是 multi-factor 的换衣服版本，已被 Q063 Phase 5 否决；用 regime 重新包装可能 sample chasing
3. Non-threshold (P3) 工程量大且 overfit 风险高；如果未来要做，应该作为独立 SPEC 而非 Q069 phase
4. Current production hard 55 gate 已被四轮研究证明 robust——这本身是有价值的发现

但我也愿意做 B 或 C 如果你想推进。**重要的是知道这是 exploration 不是 expected improvement** —— 不能用"sample chasing"的方式 frame 任何 P2/P3 结果。

### 如果选 B (regime detection)

需要 PM 先回答：
- 什么是 "regime"？基于 VIX level + VIX std + VIX direction 的硬规则？或 unsupervised clustering？
- "Recent low-vol clustered" 如何 real-time 识别（不依赖 lookahead）？
- 三个 regime 的阈值如何设？每 regime 独立 backtest？

这些是 P2 启动前 PM 应该回答的，否则 Quant 自己设计容易 overfit。

### 如果选 C (non-threshold)

最简化的 P3 设计：
- 取 last N 个相似 IVP context 的 historical bad-trade rate
- 当前 day 的 "实证 block probability" = 那个 base rate
- 如果 prob > threshold → block

这本质是 nearest-neighbor regression on historical outcomes，需要：
- 定义 "similar context" 的距离函数
- 防 overfit 的 cross-validation
- 工程量 ~1 周
