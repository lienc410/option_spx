## P0 — Decision Criteria (defined before any code)

Research question: should the V2f ES Sell Put backtone (run_phase2_v2f, SPEC-095/097) be augmented with IVP/VIX-environment entry gates derived from Q041 T1's edge thesis?

**Promote integrated ES Sell Put** if and only if one of:

1. **Ann ROE improvement** ≥ baseline + 0.3pp AND MaxDD / stress SPAN does not worsen
2. **Ann ROE flat** (±0.1pp) AND MaxDD or worst 3m cluster improves by ≥ 2pp

**Veto conditions (any one fails)**:

- V1 veto: worst single trade > −15% NLV
- Bootstrap sig_rate < 80% (block=250, 20 seeds)
- Peak SPAN > 30% NLV (at $500k NLV, ~$150k SPAN ceiling)

**Stop conditions** (per phase):

| Phase | Stop if … |
|---|---|
| P1 | IVP bucket spread < 5% avg PnL AND stop-rate spread < 5pp → no gate edge in V2f context, close P2–P5 |
| P2 | No gate has `ann_roe ≥ baseline` AND `V1 pass` → stop P3–P5 |
| P4 | `no_stop` worst trade is not materially worse than `stop_15` (no V2f-specific tail edge from STOP) → note in memo |

**Bias check**: 2nd Quant explicitly warned NOT to start from "IVP 43–55 looked good in Q041 T1, port it". Phase 1 is the unbiased question: in V2f's actual entry distribution, do IVP/VIX environment buckets exhibit different edge characteristics? Only data-driven gates proceed to P2.

## Closing notes — research framing

### What the data said vs what the prompt assumed

The original prompt framed this study as "should Q041 T1's IVP 43-55 gate be ported to V2f". Following the 2nd Quant's explicit instruction to do entry-attribution first (P1) before any gate test, the data revealed two things:

1. **Q041's IVP 43-55 gate (G3) catastrophically fails in V2f context**: ann_roe drops from 1.04% to 0.07% (−0.98pp), entry frequency collapses to 26%, sharpe 0.15 → 0.04. The hypothesis is **rebutted** as a literal port. The IVP 43-55 trades that survive ARE positively-skewed (sig=100%) but they leave 97% of V2f's annual edge on the table.

2. **A better gate emerged from the data**: VIX absolute ≥ 22 (G6). It does not appear in Q041's framework (Q041 keys off IVP relative percentile). It captures the same intuition (premium-rich regimes have higher edge) but via absolute level rather than relative ranking. The asymmetry: VIX ≥ 22 captures the HIGH_VOL regime where IV has already mean-reverted upward, leaving a clean post-fear-spike entry window.

### Significance vs production fragility

V2f_base itself has **bootstrap sig_rate = 0%** at block=250 / 20 seeds. The 725-trade pnl series is not significantly different from zero. This is a non-trivial finding about the production baseline.

V2f + G6 has bootstrap sig_rate = **100%**. The gate is not augmenting an already-significant strategy — it is **creating** statistical significance where none existed. This makes the promotion case stronger, not weaker, because the gate is doing real work.

### Practical concerns

1. **Capital under-utilization**: G6 occupies only 21% of trading days. For 79% of time, NLV sits idle. This is operationally a different strategy than V2f — closer to "wait for HIGH_VOL, then deploy". The user must accept lower aggregate capital deployment.
2. **Sample size for G6**: 146 trades over 26 years = ~5.6 trades/year. Each individual entry carries more statistical weight. The 100% bootstrap sig at block=250 is robust on this thin sample because mean PnL per trade is large.
3. **The redundancy of STOP=15 under G6** is an operational simplification opportunity for FUTURE work; keep the stop in place for now.

- Live monitor 12-month obligation: re-run P5 framework with extended data at 2027-05-14; if incremental WR < 70% on new G6 entries OR ROE turns negative, revert.
