## Rule 6 — Q042 Sleeve A 双口径报

**Sleeve-only ROE** = sleeve capital ($X) 的年化收益, e.g. 9.94% on Sleeve A capital base.

**Combined-NLV contribution** = sleeve PnL / combined NLV ($894k), e.g. ~1% if Sleeve A 用 10% capital.

**P1 ROE bridge 必须用 combined-NLV contribution** (不是 sleeve-only), 因为 ROE bridge 是 *combined account-level* 视角:

```
cash baseline 4.3%
+ SPX BPS combined-NLV contribution
+ HV Ladder combined-NLV contribution
+ Q042 Sleeve A combined-NLV contribution    ← 不是 sleeve-only 9.94%
+ Q042 Sleeve B combined-NLV contribution    ← research-only
+ V3-A marginal alpha (per Rule 1)
- friction
- idle period yield gap (vs cash baseline)
= Total Account Return
```

所有 strategy 表都必须分两列报: sleeve-only + combined-NLV contribution. PM 决策看 combined-NLV.

## Rule 7 — Operational score weighting + 用途

**Weighting (PM lock)**:
- failure mode severity: 40%
- execution complexity: 25%
- daily attention: 20%
- automation readiness: 15%

**Usage rules**:
- ✅ Operational score 用于 **promotion / demotion 讨论** (decide which strategies are production-ready vs paper-ready vs shadow-only based on operational + ROE combined)
- ✅ Operational score 用于 **complexity vs benefit ROI 讨论** (P2C retire candidates)
- ❌ Operational score **不**进 ROE 数学优化 (P3 architecture P&L formula)
- ❌ Operational score **不**作为 hard veto (低 ROE + 高 operational cost 的策略可以 retire 但不是自动 fail)

## Decision usefulness convention (per PM modification)

baseline.csv 和后续 P1 表必须每行有 `decision_usefulness` 列, 取值:

| Value | 含义 |
|---|---|
| `usable_now` | 数据 representative, 可直接进 ROE 决策 |
| `directional_only` | 数据 indicative, 不能精确数字 anchor, 只能定方向 |
| `not_decision_grade` | 数据 sample too small / window 不代表 / 单一 outlier 主导, 必须重测 |
| `N/A` | 该 cell 不适用 (e.g. permission module 不评 standalone ROE) |

PM 看 baseline 时**只用 `usable_now` 的数字做决策**, `directional_only` 只用方向, `not_decision_grade` 不引用。
