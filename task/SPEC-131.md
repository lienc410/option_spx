# SPEC-131 — 敞口感知推荐降级（**PM ratify 2026-07-07，阈值 T=40%**）

**触发**: 7/7 系统在 PM BCD 家族并发 max loss = 流动现金 50.28% 的状态下仍推荐开新 BCD。PM 直觉"此处不该加仓"的最硬内核不是技术面，是**推荐引擎只看信号格、不看组合状态**——真 gap。

## 规则（行为变更，需 PM ratify）

- 每日推荐生成时计算目标策略**家族并发 max loss / 流动现金**（复用 SPEC-129 entry-risk 单一真值函数）
- ≥ **阈值 T**（默认候选 40%，PM 定）→ 推荐卡与晨报推送**降级**：策略照常显示，但行动语气改为"**条件满足，敞口已满**（家族并发 $X = Y% 现金 ≥ T）——非加仓号召"；ACTION 类降为 STATE 语气
- **不拦**：手动开仓照常（SPEC-123 advisory 已有 cash 预算提示线）；不改 selector 信号逻辑——这是显示/推送层，不是新门
- 阈值与分母（流动现金）定义在推送文案中显式给出（今日尺度绝对值原则）

## 与既有体系边界

- 与 SPEC-079（comfortable-top）本质区别：079 是**市场状态**否决（Q087 A4 证零保护价值）；本规则是**组合状态**降级——不预测市场，只陈述账户事实
- 与 DEFERRED #20（credit 并发 cap）互补：#20 是硬 cap 治理决定，本规则是软降级，先行可独立存在

## AC 要点

阈值单测（±边界）；分母不可用 fail-soft（照常推荐 + 标注 n/a）；推送走 gateway 且类别正确；晨报与推荐卡文案一致性；零信号逻辑变更断言（selector 输出 bit-identical）。
