### 2.2 `short=0.16 / long=0.08` 是 HC selector 自创建以来的写法

证据：

```
$ git log --all --oneline --diff-filter=A -- strategy/selector.py

```

该首次提交的 diff 里包含：

```python

    Leg("SELL", "CALL", 45, 0.16, "Short call wing — HV"),
```

且这一 `0.16 / 0.08` 模式在 SPEC-064（aftermath bypass）、SPEC-066（cap=2 multi-slot）、SPEC-070 v2（delta-based 长腿）期间从未被改动，覆盖 HC `selector.py` 中 5+ 个 cell：

aftermath path 才是 `SPEC-071` 这次唯一改 short delta 的位置（line 595–601、718–724）：

- pre-SPEC-071 aftermath：`SELL 0.16 / BUY 0.08`（与 non-aftermath 一致）
- post-SPEC-071 aftermath：`SELL 0.12 / BUY CALL 0.04 / SELL 0.12 / BUY PUT 0.08`

### 2.3 因此真实结论

- HC 没有 `0.16 baseline anchor 错误`：HC 历史上从来没有过 `0.12 / 0.06` 的 IC_HV 配置
- HC 没有把 `Q026 V1` sim 的数值当成 production：HC 的 `0.16` 来自 `f119e99` 这个 selector 创建 commit，远早于 Q026 体系
- `0.16 → 0.12` 在 HC SPEC-071 的 aftermath path 是 **真实代码变更**，并且配上 `0.08 → 0.04` 长 call 的更大跨度
- 这解释了 HC 与 MC 的 cascade magnitude 差异（HC 看到 ~-$3,500/aftermath，MC 看到 ~-$554）：HC pre-SPEC-071 baseline 与 MC pre-SPEC-071 baseline 在 selector 层面就不同
