E-Trade Re-Auth

E-Trade access tokens expire at midnight ET. Use this page to refresh the token without SSH or command line.
CHECKING Checking E-Trade token status…
E-Trade 显示错误？ — token 可能已过期或未提前登录。 常见原因：① 点链接前未登录 etrade.com ② 超过 5 分钟未完成 ③ 重复使用了同一链接
Step 1
Get authorization link from E-Trade
生成新的 request token（有效期 5 分钟）。
⚠ 请先在浏览器中登录 etrade.com，再点击下方链接 — 登录页面的跳转会消耗 token 时间。

点击链接 → E-Trade 页面点 Accept → 复制 5 位 verifier code → 粘贴到下方
如果 E-Trade 显示「logon delay」错误 → 点下方「重新获取链接」按钮
Step 2
Paste verifier code
E-Trade 授权后会显示一个 5 位 code（字母+数字）。立即粘贴 — request token 5 分钟有效。
输入 verifier code 后按回车或点击按钮
Why this is necessary daily
E-Trade access tokens 设计上 每日午夜 ET 强制过期，与活跃度无关。 renew_access_token 只能处理 2 小时 inactivity timeout，无法跨越午夜边界 — 这是 E-Trade API 设计，不是 bug。

Cron 在每天 06:00 ET 自动检测 token，过期会通过 Telegram bot 发送提醒，附本页链接。
