# Q041 — Large-Cap Equity Option Income Overlay
## 策略总结 + Phase 1 设计要点

**日期：** 2026-05-03  
**阶段：** Gate 0 PASS WITH CONSTRAINTS → Phase 1 Design  
**用途：** Quant Review 参考文档

---

## 一、策略定位

Q041 是一个**独立 equity-income sleeve**，与当前 SPX 指数期权主策略（theta 类 IC/BPS/BCS）平行运营，不并入主策略代码主线。

**核心目标：** 在大盘权益持仓的基础上，通过系统性期权溢价收割提升账户 ROE，同时保持可量化的风险边界。

**账户背景：** Portfolio Margin 账户，约 $125k+，主策略已占用部分 BP。Q041 须在剩余 BP 约束下定量设计 sizing。

**与主策略的关系：**
- 资本竞争关系：Q041 与 SPX 主策略共用账户 BP，未来需要做账户级 ROE 比较
- 当前阶段不做联合优化，Q041 单独评估

---

## 二、研究范围：三个模块

经 Quant 理论筛选确认，以下三个模块具备机构化研究价值：

### Module A — Mega-Cap Covered Call Overlay
在持有大盘权益仓位的前提下，系统性卖出 OTM call，收取时间价值。

- **Alpha 来源：** Volatility risk premium（期权市场系统性高估已实现波动率）+ theta decay
- **风险：** 标的大涨时 upside 被 cap；earnings 附近 gap risk
- **Benchmark：** CBOE BXM Index（S&P 500 Buy-Write，月度 ATM CC）
- **变体：** 个股级 CC（whitelist 逐一操作）vs 指数级 CC（QQQ / SPX）

### Module B — Cash-Secured Put on High-Quality Whitelist
在严格筛选的高质量名单上系统性卖出 OTM put，以期权溢价为回报，以被动建仓为次生目标。

- **Alpha 来源：** Put skew 溢价（put IV 系统性高于 call IV at same delta）+ theta
- **风险：** 标的大幅下跌时持仓亏损；被指派时承受浮亏持仓
- **Benchmark：** CBOE PUT Index（S&P 500 PutWrite）
- **与主策略关系：** 逻辑与 SPX short put 类似，区别在于标的是个股，无 index level liquidity

### Module C — Defined-Risk Earnings Short-Vol（IV Crush）
在财报前卖出 credit spread，捕捉财报后 implied volatility 崩塌（IV crush）所释放的时间价值。

- **Alpha 来源：** 期权市场系统性高估财报事件波动幅度（implied move > realized move 在历史上显著）
- **结构：** Credit spread（call spread 或 put spread 或 iron condor），**非裸卖**
- **时序：** 财报前 5–10 天入场，财报后次日或当日收盘前平仓
- **风险：** 单日跳空超出 spread 宽度；spread 宽度设计不当时 max loss 过大
- **Benchmark：** 单腿 ATM straddle 持有至财报后一日的 P&L（衡量 defined-risk 结构相对裸卖的成本）

**明确排除：**
- 裸卖期权（naked short options）
- 单名股票极端事件押反转
- 非结构化的"月收租金"策略

## 四、Phase 1 设计要点

### 4.1 共通设计原则

**Greeks 处理方案（历史段）：**
- 使用 Barone-Adesi-Whaley (BAW) 近似计算美式期权 IV 和 delta（个股）
- SPX 使用标准 Black-Scholes（欧式）
- 过滤条件：`volume > 0`，`|moneyness| < 30%`，`DTE 7–180`
- IV 无效范围（IV < 1% 或 IV > 300%）置 NaN，不入建模

**标的价格来源：**
- SPX / QQQ：已有 `data/market_cache/yahoo__GSPC__max__1d.pkl`
- 个股（AAPL 等）：需补充 Yahoo Finance 日线收盘价序列

**指标标准包（每个模块回测结果必须包含）：**
- PnL / BP-day（主要 ROE 指标）
- Marginal $/BP-day
- Win rate、avg win、avg loss
- Worst trade、disaster window（连续最大亏损窗口）
- Max BP%（最大 buying power 占用）
- CVaR 5%（尾部风险）
- Sharpe（年化）
- 与 Benchmark 的超额回报

---

### 4.2 Module A — Covered Call 设计要点

**核心研究问题：**
1. 最优 delta 目标（20 delta vs 30 delta vs ATM）对 risk-adjusted return 的影响
2. 最优 DTE 入场（21 DTE vs 30 DTE vs 45 DTE）
3. 固定滚动（每月机械）vs 条件滚动（IV 环境过滤）哪个更优

**参数空间（Phase 1 扫描范围）：**

| 参数 | 候选值 |
|---|---|
| Target delta | 0.20 / 0.25 / 0.30 |
| Entry DTE | 21 / 30 / 45 |
| Roll DTE | 10 / 14 / 21（距到期 DTE 触发滚动）|
| Universe | 全 10 个大盘个股 / SPX / QQQ |
| IV filter | 无过滤 / IV > 20% 才入场 |

**验收基准：**
- 与 BXM 相比，是否显著超额（Sharpe ≥ BXM + 0.1）
- 2022 熊市段是否守住（不明显劣于 BXM）

---

### 4.3 Module B — Cash-Secured Put 设计要点

**核心研究问题：**
1. Put skew 溢价在个股上是否稳定（历史上 put IV 是否系统性高于同 delta call IV）
2. 入场条件：无条件机械卖 vs 只在 IV 高时卖（IV Rank > 50%）
3. 被指派后是否转 CC（wheel strategy）vs 直接平仓

**参数空间：**

| 参数 | 候选值 |
|---|---|
| Target delta | 0.20 / 0.25 / 0.30 |
| Entry DTE | 21 / 30 / 45 |
| IV filter | 无 / IVR > 30% / IVR > 50% |
| Assignment handling | 平仓 / 转 CC |

**验收基准：**
- 与 PUT Index 相比的超额表现
- 个股 CSP vs SPX CSP 的风险调整回报比较

---

### 4.4 Module C — Earnings Short-Vol 设计要点

**核心研究问题：**
1. 历史上 implied move（财报前隐含的单日预期波动）vs realized move（实际波动）的溢价率是否显著且稳定
2. 最优 spread 宽度：0.5× / 1× / 1.5× implied move
3. 最优入场时间：财报前 1 天 vs 3–5 天 vs 7–10 天（IV 建立阶段）
4. 方向选择：双向 iron condor vs 单向 put spread（考虑 put skew）

**数据需求（额外）：**
- 历史财报日期：可从 `yfinance.Ticker(sym).get_earnings_dates()` 拉取
- 财报前后 IV 序列：历史段需 BAW 反推；2026-05-03 后 Schwab 直接提供

**参数空间：**

| 参数 | 候选值 |
|---|---|
| Entry DTE（相对财报） | T−1 / T−3 / T−7 |
| Exit timing | T+1 开盘 / T+1 收盘 |
| Spread width | 0.5× / 1.0× / 1.5× implied move |
| Direction | Put spread / Call spread / Iron condor |
| Universe filter | 全 whitelist / 仅流动性 top-5 |

**验收基准：**
- implied move premium 显著性（t-stat > 2，样本量 ≥ 50 events）
- 剔除 2022-04 / 2025-04 等极端事件后结论是否稳健
- CVaR 5% 不超过单次 spread max loss 的 3×（确认无系统性 gap 风险）

## 五、Phase 1 执行顺序建议

| 顺序 | 模块 | 理由 |
|---|---|---|
| 1st | **Module C（Earnings）** | 样本独立性最强（每季一次），结论最干净；先确认"结构性定价偏差是否存在" |
| 2nd | **Module A（Covered Call）** | 数据最完整，BXM benchmark 清晰 |
| 3rd | **Module B（CSP）** | 与 Module A 共享大量基础设施，放在 A 之后效率最高 |

## 六、开放问题（提请 Quant Reviewer 关注）

1. **4 年窗口偏差**：2022-05 起点恰在高点，含一次大熊市（2022）和强牛市（2023–2025）。结论的时间稳定性如何验证？
2. **美式期权 early exercise**：BAW 对 deep ITM call（接近到期的高股息股）是否足够精确？
3. **Volatility surface**：本设计仅用 ATM IV，不建 vol surface。对于 put skew 分析（Module B）和 spread 定价（Module C），是否需要 moneyness-specific IV？
4. **个股相关性**：17 个标的间高度相关（科技股集中）。在熊市中 CC/CSP 的同步被指派风险如何度量？
5. **BP 竞争**：Q041 与 SPX 主策略共用 PM 账户 BP。在高 VIX 环境下 SPAN margin 扩张时，哪个策略优先占用 BP？
6. **交易成本**：历史数据无 bid/ask，回测用 mid price 低估实际成本。对于流动性较低的标的（ASML、TSM），spread cost 是否实质性侵蚀 alpha？
